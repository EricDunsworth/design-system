Adding structured data on Canada.ca pages - Pilot instructions

From: Treasury Board of Canada Secretariat

Adding structured data to a webpage can help deliver better answers to Canadians.

On this page

  • What is structured data
  • How to add structured data
  • Schema code and examples
  • How to measure impact of structured data
  • Additional resources

What is structured data

Structured data is a way to tag content in a “machine-readable” way.

It allows you to add meaning to the page.

It should help deliver:

  • better and more informative search results (often in Google and Bing)
  • better voice search results (for services like Alexa and Google Home)

Note that you can't control how search results appear (for example, what appears in the featured snippet). Adding structured data may or may not change search results.

Structured data needs to be a “true representation” of the page. You can’t add structured data about content that is not covered in the page.

Adding structured data is just one of the techniques you can use to optimize your content for search. Improving the actual content on the page is the first step to take.

  • Google: Understand how structured data works
  • Google: Structured data guidelines
  • Optimize your content for voice search

How to add structured data

Step 1: Determine the most effective schema to add

There are multiple structured data schemas you can use to add meaning to your page.

Consult the list of schemas on Schema.org or Google, or look at the examples provided on this page.

  • Schema.org schemas
  • Google structured data
  • Schema code and examples for Canada.ca

Step 2: Choose the method

You can use 2 different methods to add structured data to a Canada.ca page: JSON-LD and RDFa. You can also use a combination of both methods.

The documentation for each schema usually provides the markup for each method.

Adding JSON-LD script

  • script is added as a code snippet separate from the content of the page
  • sample code can be copied/pasted and values adjusted
  • requires content duplication
  • risk of conflict with the structured data already embedded in WET templates
  • method recommended by Google

Use a validator, such as JSONLint, to make sure your code is valid.

Adding JSON-LD in Adobe Experience Manager (AEM)

If your page is published in AEM, you can add structured data in JSON-LD by using the structured data component, in the page properties.

Follow instructions in GCPedia (only available on the GC network).

Adding RDFa code

  • attributes are added to individual elements on the page
  • attributes may require additional elements to be added to the code to create structure, such as section, div and span
  • no duplication of content
  • risk of conflict with the structured data already embedded in WET templates

Step 3: Test the structured data code

You can copy code snippets or full URLs to test code with different tools:

  • Rich results test: shows which rich result types (supported by Google) were found on the page, as well as any errors or suggestions for your structured data
  • Structured data test: validates the structured data - this tool is being deprecated by Google

Both tools should yield the same result. By pasting the URL or the code of the page, you can see if it’s valid, and if the values assigned to different fields are the correct ones.

Step 4: Validate HTML code and check accessibility

Before you publish the page with structured data, make sure that the HTML is valid and the page is accessible. You can use online validators to help you with this:

  • W3C HTML validator
  • WAVE Web Accessibility Evaluation Tool

Step 5: Publish the page, reindex and monitor

Once your structured data is published, ask Google and Bing to reindex the page.

Check the status of your code in the Canada.ca Google Search Console.

How to measure impact of structured data

You can measure the impact of structured data looking at the Google Search console. You can get statistics on how the rich snippet is performing:

  • what’s the proportion of the clicks to a page generated from a rich snippet
  • what are the search queries that generate rich snippets
  • is the click-through rate different with a rich snippet

Adding UTM codes to hyperlinks in the rich data can also help you understand when people are clicking in the links in the rich snippet.

Schema code and examples

SpecialAnnouncement

When to use

The specialAnnouncement schema is a new experimental schema developed for announcements related to COVID-19. It's mainly for health related information (symptoms, quarantine, testing, etc.), but also for closures, reopenings, and special government benefits.

Official guidance

  • Schema.org specialAnnouncement guidance
  • Google specialAnnouncement guidance

Code

JSON-LD
  • Add id attribute to the JSON-LD HTML script element named “wb-script”.
  • Use the same JSON-LD root id anchor as the one for the main content of the page. Both will use “#wb-main” to specify semantically they are describing the same subject.
<script id="wb-script"
type="application/ld+json">
 {
  "@context":"http://schema.org",
  "@type":"SpecialAnnouncement",
  "@id": "#wb-main",
  "mainEntityOfPage":{

FAQPage

When to use

Use to add a set of “questions and answers” (or a set of headings and their respective content). Either tag the visible content (using RDFa), or add a layer in JSON-LD.

Use on:

  • Service initiation pages
  • Guidance documents
  • Actual FAQs, though FAQ pages are strongly discouraged on Canada.ca in most cases - read our blog post: Avoiding FAQs, with one exception

Official guidance

  • Schema.org FAQPage guidance
  • Google FAQPage guidance

Speakable

When to use

Use to identify any section of content that would be best suitable for text-to-speech: the part of the page that encapsulates the most important content people are looking for on that page.

Can be used on any page.

Official guidance

  • Schema.org Speakable guidance
  • Google Speakable guidance

Code

JSON-LD
  • Add id attribute to the JSON-LD HTML script element named “wb-script”.
  • Use the same JSON-LD root id anchor as the one for the main content of the page. Both will use “#wb-main” to specify semantically they are describing the same subject.
<script id="wb-script"
type="application/ld+json">
 {
  "@context":"http://schema.org",
  "@id": "#wb-main",
  "@type": "WebPage",
  "name": "How much Canada Child Benefit you could get",
  "speakable":{
   "@type": "SpeakableSpecification","cssSelector": [".speakable1"]
   },
  "url":"https://www.canada.ca/en/revenue-agency/services/child-family-benefits/canada-child-benefit-overview/canada-child-benefit-we-calculate-your-ccb.html"}
</script>
<body vocab="http://schema.org/" typeof="WebPage">
 <div typeof="Organisation" resource="#wb-publisher">
  <img src="/link/to/a/logo" alt="Publisher name" property="logo" />
  <meta property="name" content="Publisher name" />
 </div>
