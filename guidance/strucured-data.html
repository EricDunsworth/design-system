Adding structured data on Canada.ca pages - Pilot instructions

From: Treasury Board of Canada Secretariat

These instructions are interim guidance to help piloting adding structured data to Canada.ca pages. It's not an official guidance for Canada.ca.

With this pilot, we're trying to better understand how structured data can help people find the answer they're looking for and complete their task.

On this page

  • What is structured data
  • How to add structured data
  • Schema code and examples
  • Additional resources

What is structured data

Structured data is a way to tag content in a “machine-readable” way.

It allows you to add explicit meaning to the page.

It should help deliver:

  • better and more informative search results (mostly in Google and Bing)
  • better voice search results

While adding explicit meaning helps third-party provide better search results, it's important to note that you can't actually control the design of the search results on Google or Bing, for example. Adding strucutured data may or may not change search results.

Structured data needs to be a “true representation” of the page. You can’t add structured data about content that is not covered in the page.

Adding structured data is just one of the techniques you can use to optimize your content for search. Improving the actual content on the page is first step to take.

  • Google: Understand how structured data works
  • Google: Structured data guidelines
  • Optimize your content for voice search

How to add structured data

Step 1: Determine the schema you want to add

There are multiple structured data schemas you can use to add explicit meaning to your page.

Consult the list of schemas on Schema.org or Google, or look at the examples provided on this page.

  • Schema.org schemas
  • Google structured data
  • Schema code and examples for Canada.ca

Step 2: Choose the method

You can use 2 different methods to add structured data to a Canada.ca page: RDFa and JSON-LD. You can also use a combination of both methods. The documentation for each schema usually provides the markup for each method.

Adding RDFa code

  • tags are added directly within the HTML content
  • slighty more complex to code than JSON-LD - you need to add tags in the right places, rather than just pasting a script and adjusting values
  • involves less duplication of content than JSON-LD
  • less risk of conflict with the structured data generated by the CMS

Adding JSON-LD script

  • script is added separately from the HTML content
  • slighty less complex to code than RDFa, as you can paste a script and adjust values, rather than adding tags in the right places in the code
  • involves more duplication of content than RDFa
  • increased risk of conflict with the structured data generated by the CMS
  • method recommended by Google

Step 3: Test the structured data code

You can copy code snippets or full URLs to test code with different tools:

  • Rich results test
  • Structured data test

Both tools should yield the same result. By pasting the code, you can see if it’s valid, and if the values assigned to different fields are the correct ones.

Step 4: Validate HTML code and check accessibility

Before you publish the page with structured data, make sure that the HTML is valid and the page is accessible. You can use online validators to help you with this:

  • W3C HTML validator
  • WAVE Web Accessibility Evaluation Tool

Schema code and examples

SpecialAnnouncement

When to use

The specialAnnouncement schema is a new experimental schema developed for announcements related to COVID-19. Mainly for health related information (symptoms, quarantine, testing, etc.), but also for closures, reopenings, and special government benefits.

Official guidance

  • Schema.org specialAnnouncement guidance
  • Google specialAnnouncement guidance

Code

JSON-LD
  • Add id attribute to the JSON-LD HTML script element named “wb-script”.
  • Use the same JSON-LD root id anchor as the one for the main content of the page. Both will use “#wb-main” to specify semantically they are describing the same subject.
<script id="wb-script"
type="application/ld+json">
 {
  "@context":"http://schema.org",
  "@type":"SpecialAnnouncement",
  "@id": "#wb-main",
  "mainEntityOfPage":{
   "@type":"WebPage",
   "@id":"https://www.canada.ca/en/public-health/services/diseases/coronavirus-disease-covid-19.html"
   },
  "name":"Coronavirus disease (COVID-19): information from the government of Canada",
  "text":"Coronavirus disease (COVID-19) information for Canadians including links to disease updates, travel advice, how to be prepared, symptoms, prevention, risk, Canada's response, current case numbers, answers to questions and links to printable resources.",
  "spatialCoverage": {
   "@type": "Country",
   "name": "Canada"
   },
  "publisher":{
   "@type":"GovernmentOrganization",
   "@id":"#wb-publisher",
   "name":"Government of Canada",
   "url":"https://www.canada.ca/en.html"
   },
  "category":"https://www.wikidata.org/wiki/Q81068910",

  "quarantineGuidelines":"https://www.canada.ca/en/public-health/services/diseases/2019-novel-coronavirus-infection/prevention-risks.html",

When to use

Use to add a set of “questions and answers” (or a set of headings and their respective content). Either tag the visible content (using RDFa), or add a layer in JSON-LD.

Use on:

  • Service initiation pages
  • Guidance documents
  • Actual FAQs

Official guidance

  • Schema.org FAQPage guidance
  • Google FAQPage guidance

Speakable

When to use

Use to identify any section of content that would be best suitable for text-to-speech: the part of the page that encapsulates the most important thing people are looking for on that page.

Can be used on any page.

Official guidance

  • Schema.org Speakable guidance
  • Google Speakable guidance

Code

JSON-LD
  • Add id attribute to the JSON-LD HTML script element named “wb-script”.
  • Use the same JSON-LD root id anchor as the one for the main content of the page. Both will use “#wb-main” to specify semantically they are describing the same subject.
<script id="wb-script"
type="application/ld+json">
 {
  "@context":"http://schema.org",
  "@id": "#wb-main",
  "@type": "WebPage",
  "name": "How much Canada Child Benefit you could get",
  "speakable":{
   "@type": "SpeakableSpecification","cssSelector": [".speakable1"]
   },
  "url":"https://www.canada.ca/en/revenue-agency/services/child-family-benefits/canada-child-benefit-overview/canada-child-benefit-we-calculate-your-ccb.html"}
</script>
<body vocab="http://schema.org/" typeof="WebPage">
 <div typeof="Organisation" resource="#wb-publisher">
