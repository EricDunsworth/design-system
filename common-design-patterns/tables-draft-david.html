Tables: Canada.ca design system

  • Last updated: 2021-04-19

Data tables are a means for efficiently communicating data in rows and columns.

On this page

  • When to use
  • What to avoid
  • How to implement
  • Research and blog posts
  • Latest changes
  • Discussion

When to use

  • Use to present data
  • Use to organize related information, such as contact numbers and details

What to avoid

  • Do not use data tables to force layout for text
  • Never include tabular data as an image

How to implement

There are 2 main types of tables:

  1. Basic tables
  2. Responsive tables (beta)

Basic tables

Use basic tables when the content is not likely to be consumed on mobile.

Resonsive tables - Beta

Responsive provisional tables are:

  • only supported on simple tables
  • not supported by IE 11

Responsive cards work with or without a filter. Add the class wb-tables and any data-wb-tables="" values you require to the <table>.

Customizing responsive tables

Option Description How to use
Left aligned Align the text in any cell on mobile cards Add the class text-left to any <td>
Small text Make text smaller in mobile cards Add the class small the <table>.

Research and blog posts

We tested the use of responsive tables successfully during optimization projects with Health Canada for recalls and safety sorting, CRA for organizing contact us information, and across various COVID-19 optimizations. Responsive tables make it easier for users on mobile devices to use without having to scroll to the side to see all the content.
